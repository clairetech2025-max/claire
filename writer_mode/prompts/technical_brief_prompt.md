# Technical Brief Prompt

Create a partner-facing markdown brief.

Requirements:

- Plain-English explanation first.
- Named architecture second.
- No hype-only language.
- No confidential source code.
- No patent claim over-disclosure.
- No invented performance claims.
- No military, government, or partner claims unless present in source material.
- Include `Draft for human review`.

Voice:

- Clear.
- Serious.
- Concrete.
- Buyer-readable.
- Technical without becoming theatrical.

Structure:

1. Plain-English explanation.
2. Problem.
3. What Claire does.
4. Named architecture.
5. Evidence or demo artifact.
6. Boundaries and review notes.
