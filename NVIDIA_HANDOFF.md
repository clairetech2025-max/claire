# NVIDIA Handoff

CLAIRE is a governed runtime around model intelligence.

ARE provides chronological memory authority. The model can read admitted memory context, but it does not own memory.

Trace provides audit evidence. Normal traces retain hashes, lane, risk, validation metadata, and memory-write status without exposing private payloads.

Sentinel validates output before release and blocks unsafe or cross-lane behavior.

Veritas and CourtListener are governed subsystem pressure tests:

- Veritas validates financial intelligence and trading-monitoring boundaries.
- CourtListener validates legal-monitoring and source-backed research boundaries.
- Neither subsystem is a memory authority.

NVIDIA Nemotron, NIM, and NeMo Guardrails can strengthen model serving, inference scale, and guardrail layers without replacing ARE.

## Current Proof Targets

- CLAIRE answers through `ClaireRuntime`.
- Normal answers hide lane, risk, trace, Sentinel, context packet, and routing notes.
- Horse/hoof/equine prompts route to `HORSE_STEWARDSHIP`.
- Crypto/Kraken/Veritas prompts route to `TRADING_STATION`.
- CourtListener prompts route to `LEGAL_CASE`.
- Live trading is blocked from normal chat.
- Passphrase-like text typed into chat is redacted from answers, memory writes, and raw trace payloads.
- Trace records reproducible audit evidence.

## NVIDIA Fit

- Nemotron/NIM: replaceable model brain downstream of the governed runtime.
- NeMo Guardrails: optional guardrail layer for visible-output policy and unsafe-request interception.
- ARE: chronological memory authority that should remain independent of model serving.
- Trace: reproducibility and audit layer for demos, validation, and enterprise review.

## Language Standard

Use technical, reproducible language:

- repo
- branch
- commit
- commands
- validation output
- trace id
- lane metadata
- blocked action proof

Avoid hype, unsupported claims, and public-facing claims that have not been approved.
