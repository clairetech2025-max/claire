# CODEX CLAIRE GYRO PHASE PLAN

## Current Defect

CLAIRE sometimes answers architecture, benchmark, pipeline, and ordinary capability questions with stale session fragments, unrelated memory, document text, or canned architectural speeches. The issue is execution authority: memory/session/document/context handlers can influence or replace the answer before canonical orientation and memory eligibility complete.

## Current Authoritative Call Graph

`POST /ask or /reply -> build_reply(q) -> recent session context before classification -> many special-case handlers -> optional classify_query -> optional ARE recall/gating -> contextualize_prompt may inject session/durable/Gyro memory -> query_llm posts to GO backend -> cleanup -> trace/session/durable/TMF writes`

## Target Call Graph

`input -> SignalEnvelope normalization/lineage -> provisional Q Insight -> C3RP lane/authority/route classification -> memory eligibility -> optional governed ARE/vector recall -> FARE projection -> Sentinel inspection -> Diode/WriteBarrier -> GO generation -> output validation -> Trace/TrailLink/Ledger -> approved writeback`

## Phase One Repair

Objective: establish one authoritative backend execution path and stop contaminated context from appearing in final answers.

Repair:

- Add or designate one backend route function for normal chat.
- Classify before any recall or session/document prompt context.
- Add explicit memory modes: OFF, SUPPORT, STRICT, REQUIRED, QUARANTINED.
- Demote canned architecture/identity/ARE/RAG/benchmark/pipeline replies from final-answer authority.
- Keep deterministic hardcoded text only for errors, status, safety notices, empty states, explicit UI copy, and protocol messages.
- Keep GO as the normal generation backend.
- Do not change GUI face.

## Full Gyro Phase Sequence

1. Phase One: authoritative execution path and bypass removal.
2. Phase Two: Q Insight state object and lifecycle.
3. Phase Three: sparse 360 x 360 latent coordinate space.
4. Phase Four: coordinate binding, release, rebinding, expiration, and quarantine.
5. Phase Five: continuous motion, velocity, drift, contradiction pressure, and stabilization.
6. Phase Six: ARE, FAISS/cuVS, FARE, Sentinel, and Diode under one governed contract.
7. Phase Seven: text, recalled memory, and tool output as signal types.
8. Phase Eight: documents, OCR, speech, APIs, telemetry, policy events, model proposals, and other agents.
9. Phase Nine: complete trace, replay, recovery, and deterministic event reconstruction.
10. Phase Ten: benchmarks comparing raw LLM, conventional RAG, governed ARE, and Gyro-oriented ARE.
